## P0 — Critical Issues

### 1. Footer headings rendered as H3 on homepage — pollutes heading hierarchy

- **Page(s):** `/`
- **Impact:** The 4 footer labels (Oficina, Navegación, Legal, Social) appear in the crawled H3 list, inflating H3 count to 30 and confusing crawlers about page structure. Googlebot uses headings to understand content topics; navigational labels should never be headings.
- **Current state:** `<h3>Oficina</h3>`, `<h3>Navegación</h3>`, `<h3>Legal</h3>`, `<h3>Social</h3>` are emitted in the footer. Same issue likely exists on every page with the same footer, but the crawl only flags them distinctly on the homepage because the Services section also duplicates 7 H3 items (same service names appear twice, likely from a carousel with two DOM copies).
- **Suggested fix:** Change footer heading elements from `<h3>` to `<p>`, `<strong>`, or an `aria-hidden` element. For the duplicate service H3s, hide the carousel's off-screen copy from the accessibility tree with `aria-hidden="true"`.

```tsx
// Footer: replace <h3> with <p class="text-meta uppercase">
<p className="text-meta uppercase">Oficina</p>

// Services carousel: add aria-hidden to the hidden copy
<div aria-hidden="true" className="...duplicate-carousel...">
  ...
</div>
```

### 9. og:type is "article" for all service sub-pages — incorrect type

- **Page(s):** All 7 `/servicios/[slug]` pages
- **Impact:** Open Graph `type="article"` signals to social platforms that the content is a blog post or news article, not a service/business page. This affects how the URL appears when shared on Facebook/LinkedIn. It can also confuse Bing's content classification. The correct type for service pages is `"website"` or `"service"` (though OG doesn't have a service type — `"website"` is appropriate).
- **Current state:** `og:type = article` on `/servicios/seguro-salud-texas`, `/servicios/seguro-vida-dallas`, `/servicios/seguro-gastos-finales-texas`, `/servicios/seguro-dental-texas`, `/servicios/seguro-vision-texas`, `/servicios/medicare-texas`, `/servicios/seguro-indemnizacion-texas`.
- **Suggested fix:** Change the metadata export on each service page.

```tsx
// src/app/servicios/[slug]/page.tsx — inside generateMetadata
openGraph: {
  type: "website",  // not "article"
  ...
}
```
