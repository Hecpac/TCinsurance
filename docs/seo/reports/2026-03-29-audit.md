## P1 Important Issues

### P1-1. Title tags exceed 60-65 character limit on 20 of 25 pages

**Impact:** Google truncates titles in SERPs, reducing CTR and keyword visibility.

**Affected pages (worst offenders):**

| Page | Length |
|------|--------|
| `/blog/texas-hb2067-explicaciones-automaticas-seguros-2026` | 118 chars |
| `/blog/agente-seguros-salud-lewisville-tx` | 100 chars |
| `/blog/medicare-prior-authorization-texas-2026` | 96 chars |
| `/blog/subsidios-aca-2026-texas` | 95 chars |
| `/blog/seguro-gastos-finales-texas-guia-completa` | 92 chars |
| `/servicios/seguro-gastos-finales-texas` | 88 chars |
| `/servicios/seguro-indemnizacion-texas` | 88 chars |
| All 7 service detail pages | 74-88 chars |
| All 3 city landing pages | 67-71 chars |
| `/sobre-mi` | 78 chars |

Only 4 pages are within the recommended range: `/` (64), `/servicios` (64), `/blog` (19 -- too short), and the two legal pages.

**Current pattern for service pages:**
```
Seguro de Gastos Finales en Dallas, Fort Worth y Lewisville, Texas | Asesoria en Espanol
```

**Suggested fix:** Drop city enumeration from service titles. Use the brand suffix `| TC Insurance` instead of the long variant:

```tsx
// src/app/servicios/seguro-gastos-finales-texas/page.tsx
// Before:
title: "Seguro de Gastos Finales en Dallas, Fort Worth y Lewisville, Texas | Asesoria en Espanol"
// After:
title: "Seguro de Gastos Finales en Texas | TC Insurance"
```

For blog posts, drop the `| TC Insurance` suffix from the title metadata (the brand is already visible in the SERP via the site name):

```tsx
// src/app/blog/[slug]/page.tsx generateMetadata()
// Before:
title: `${post.title} | TC Insurance`,
// After:
title: post.title.length > 60 ? post.title : `${post.title} | TC Insurance`,
```

## P2 Improvements

### P2-1. Homepage canonical missing trailing slash

**Current:** `https://tcinsurancetx.com`
**Full URL:** `https://tcinsurancetx.com/`

The canonical tag on the homepage points to the non-trailing-slash version while the actual URL includes a trailing slash. Google handles this gracefully, but a mismatch can cause unnecessary redirect chains and split signals.

**Suggested fix:** In `src/app/page.tsx`, ensure the canonical uses a trailing slash, or update `src/config/site.ts` so `siteConfig.seo.siteUrl` includes a trailing slash for the root. The simplest approach:

```tsx
// In the homepage metadata
alternates: {
  canonical: `${siteConfig.seo.siteUrl}/`,
},
```

### P2-3. Duplicate H3 headings on homepage and /servicios

The homepage and `/servicios` both render the services list twice (likely a desktop/mobile duplicate), resulting in duplicate H3 elements:

- "Seguro de Salud" x2
- "Seguro de Vida" x2
- "Gastos Finales" x2
- (and 4 more service names x2 each)

**Impact:** Confusing heading hierarchy for screen readers and crawlers. Not a ranking factor per se, but a negative signal for content structure.

**Suggested fix:** Hide the duplicate set from assistive tech using `aria-hidden="true"` on the visually-hidden duplicate, or render only one set responsively:

```tsx
// If one set is mobile-only, ensure it has aria-hidden
<div className="block md:hidden" aria-hidden="true">
  {/* duplicate services for mobile layout */}
</div>
```

### P2-4. Concatenated H1 text on /servicios and /blog

**`/servicios` H1:** `"Seguros de Salud, Viday Gastos Finales en Dallas / Fort Worth / Lewisville."`
(Missing space: "Vida y" rendered as "Viday")

**`/blog` H1:** `"Blogestrategico."`
(Missing space: "Blog estrategico" rendered as "Blogestrategico")

These are likely caused by line breaks in JSX being collapsed. The crawled text shows no space between words.

**Impact:** Keywords are malformed for crawlers. "Viday" is not a recognizable keyword.

**Suggested fix:** Add explicit `{" "}` or `&nbsp;` between JSX elements, or ensure a text space exists in the source:

```tsx
// Instead of:
<h1>Seguros de Salud, Vida<br />y Gastos Finales...</h1>
// Use:
<h1>Seguros de Salud, Vida{" "}y Gastos Finales...</h1>
```
